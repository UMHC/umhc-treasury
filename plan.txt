UMHC Treasurer Mobile Improvement Plan

Goal
Improve how the website looks and functions on phones, with a specific focus on reducing unnecessary vertical space, while keeping the PC version unchanged.

Core Principle
Every change in this plan is mobile additive and desktop preserving.
- Desktop remains the default render path.
- Mobile changes are gated by narrow-screen breakpoints or explicit mobile-only UI state.
- No service, store, or backend contract changes.
- No desktop layout should be refactored unless the change is provably neutral on wide screens.

Architecture Constraint
Keep the existing design philosophy intact:
Action -> Service -> State -> UI Update

This means:
- UI changes stay in the component and styling layers.
- Business logic remains in services and logic modules.
- State remains the single source of truth.
- Mobile behavior should not introduce DOM-driven hacks that bypass component/state structure.

Acceptance Criteria
- Desktop at existing breakpoints is visually unchanged or negligibly changed.
- Phone screens are shorter, with fewer always-open controls.
- Primary actions remain visible and secondary controls are progressively disclosed.
- Phone layout width remains stable during refresh and initial load.
- Dense table-driven views have a readable phone-native presentation instead of relying on desktop tables.
- View-only tag naming is consistent in navigation and in-section headers.
- The tags timeframe control matches the established green theme.
- The architecture stays intact: action -> service -> state -> UI update.
